2. 스타일링 방식
   Q. CSS 프레임워크 선호도가 있으신가요? (예: Tailwind CSS, Bootstrap, 순수 CSS 등)
   A. MUI 선호. Bootstrap이든 순수 css든 첫눈에 이해하기 쉽고 유지보수 및 관리만 잘되면 상관없음

3. 빌드 도구
   Q. Vite, Create React App 등 선호하시는 빌드 도구가 있나요?
   A. 둘다 써봐서 상관없음.

4. 배포 환경
   이 앱은 로컬에서만 사용하실 예정인가요, 아니면 웹 서버에 배포할 계획이신가요?
   백엔드/DB 연동 없이 순수 클라이언트 사이드(LocalStorage 기반)로만 운영할 예정이 맞으신가요?

- 배포 예정이 있으나 백엔드/DB연동은 없고 순수 클라이언트 사이드(LocalStorage) 기반으로 동작. 일단 배포는 신경쓰지마. 나중에 번들링(js, html)만 잘하면 될것같아.

5. 레이아웃 구조
   Q. PRD에는 "타이머 목록 뷰"와 "간트 차트 뷰", "리포트 뷰"가 있는데, 이를 탭(Tab) 형태로 전환하는 것이 좋을까요, 아니면 한 페이지에 모두 표시하는 것이 좋을까요?
   A. 타이머 목록이 경험상 많아봤자 10개 라서 (계속 완료치니 목록에서 사라짐) 타이머 목록 뷰와 간트 차트 뷰를 같이 보여주자.
   리포트 뷰는 따로 탭으로 빼자. 사용자가 메뉴를 뺄지 한 페이지에 보여줄지를 커스텀할 수 있으면 베스트고.

7. 언어
   Q. UI에 표시되는 텍스트는 한국어로만 표시하면 되나요? (다국어 지원 불필요 확인)
