## 1. 시작하기

상단 네비게이션 바에서 **프로젝트 분석** 탭을 클릭하거나, `Alt + 4` 단축키로 이동합니다.

```
일간 타이머 | 주간 일정 | 배포 캘린더 | [프로젝트 분석] | 설정
```

## 2. 프로젝트 분석 페이지

프로젝트 분석 페이지는 기록된 업무를 **프로젝트 → 작업** 단위로 시간을 분해하여 보여줍니다.

### 기간 선택
- **월간**: 해당 월의 전체 업무를 집계합니다. 좌우 화살표로 이전/다음 달 이동.
- **분기**: 3개월 단위로 집계합니다.
- **직접 지정**: 시작일~종료일을 직접 선택하여 원하는 기간을 조회합니다.

### 프로젝트 필터
- 우측 검색창에서 특정 프로젝트를 선택하면 해당 프로젝트의 작업만 표시됩니다.
- 비우면 전체 프로젝트를 볼 수 있습니다.

## 3. 시간 분해 보기

### 카테고리별 비율 바
페이지 상단에 전체 업무의 카테고리(분석, 개발, 회의 등) 비율이 스택 바로 표시됩니다.
- 각 구간에 마우스를 올리면 카테고리명, 시간, 비율이 표시됩니다.

### 프로젝트별 작업 테이블
각 프로젝트마다 테이블이 표시되며, 다음 정보를 볼 수 있습니다:

| 열 | 설명 |
|---|---|
| 작업명 | 해당 프로젝트에서 수행한 작업 제목 |
| 카테고리 | 가장 빈번하게 사용된 카테고리 |
| 세션 | 해당 작업의 세션(기록) 수 |
| 소요 시간 | 총 소요 시간 (점심시간 제외) |
| 비율 | 프로젝트 내 비율 (%) + 바 차트 |

**사용 예시:**
> "LifeCycle 관리" 프로젝트를 선택하면 설계 28시간(28.6%), 개발 32시간(32.7%), 코드리뷰 12시간(12.2%), 리팩토링 14시간(14.3%), 테스트수정 12시간(12.2%)처럼 분해되어 표시됩니다.

## 4. 세션 드릴다운 및 메모 편집

### 드릴다운
작업 행을 **클릭**하면 해당 작업의 개별 세션 목록이 펼쳐집니다.

| 열 | 설명 |
|---|---|
| 날짜 | 세션 시작 날짜 |
| 시간 | 시작~종료 시간 |
| 소요 | 해당 세션의 소요 시간 |
| 비고 | 메모(note) — 클릭하여 편집 가능 |

### 메모 인라인 편집
- 비고 열의 "메모 추가..." 텍스트를 클릭하면 편집 모드로 전환됩니다.
- 내용 입력 후 `Enter` 또는 ✓ 버튼으로 저장합니다.
- `Escape` 또는 ✕ 버튼으로 취소합니다.
- 영역 바깥을 클릭해도 자동 저장됩니다.

**사용 예시:**
> "리팩토링이 14시간이나 걸린 이유"를 파악하기 위해, 리팩토링 행을 클릭한 뒤 각 세션의 메모를 확인합니다. "코드리뷰 피드백이 설계 수준의 변경을 요구", "DB 스키마 재설계 필요" 등의 맥락을 읽어낼 수 있습니다.

## 5. 인터럽트 분석

페이지 하단의 **인터럽트 분석** 섹션에서 특정 작업의 발생 패턴을 분석할 수 있습니다.

### 키워드 추가
1. 검색창에 추적할 키워드를 입력합니다 (예: "TP 대응").
2. `Enter`를 누르면 키워드가 태그로 추가됩니다.
3. 작업명, 카테고리, 프로젝트 코드에서 키워드가 포함된 모든 작업이 매칭됩니다.
4. 키워드 태그의 ✕를 클릭하면 제거됩니다.

### 분석 결과
키워드를 추가하면 다음 분석이 표시됩니다:

#### 요약 통계
- **총 인터럽트**: 매칭된 세션 수
- **총 소요시간**: 인터럽트에 사용된 총 시간
- **최다 발생 주**: 가장 인터럽트가 많았던 주
- **피크 시간대**: 인터럽트가 가장 집중된 2시간 구간

#### 주차별 히트맵
월~금 요일 × 주차 매트릭스로, 인터럽트가 집중된 시점을 색상 강도로 보여줍니다.
- 진한 빨강: 인터럽트 많음
- 빈 셀: 인터럽트 없음

#### 시간대별 분포
06~24시 구간별 막대 차트로, 하루 중 어느 시간대에 인터럽트가 집중되었는지 보여줍니다.

#### 발생 이력
매칭된 모든 세션을 시간순으로 나열한 테이블입니다.

**사용 예시:**
> "TP 대응" 키워드를 추가하면, 4주차에 8건·11시간이 집중되었고 피크 시간대가 14:00~16:00임을 확인할 수 있습니다. 이를 통해 QA 기간 오후에 인터럽트가 몰린다는 패턴을 발견할 수 있습니다.

## 6. 타이머 완료 시 메모 입력

일간 타이머 페이지에서 작업 **완료** 버튼을 클릭하면, 메모 입력 필드가 나타납니다.

- **저장**: 메모를 입력하고 `Enter` 또는 "저장" 버튼을 클릭합니다.
- **건너뛰기**: 메모 없이 바로 완료하려면 "건너뛰기" 버튼 또는 `Escape`를 누릅니다.

입력한 메모는 프로젝트 분석 페이지의 세션 드릴다운에서 확인할 수 있습니다.

## 7. 내보내기 (복사/CSV)

### 클립보드 복사
"복사" 버튼을 클릭하면 분석 결과가 텍스트 형식으로 클립보드에 복사됩니다.
```
[2026년 3월] 프로젝트 분석

■ LifeCycle 관리 — 총 98시간
  ├ 개발 [개발] — 32시간 (32.7%)
  ├ 설계 [분석] — 28시간 (28.6%)
  ...

총 업무시간: 98시간
```

### CSV 다운로드
"CSV" 버튼을 클릭하면 프로젝트별 작업 상세가 CSV 파일로 다운로드됩니다.
Excel에서 열어 추가 분석에 활용할 수 있습니다.

## 8. FAQ

**Q. 프로젝트 분석에 표시되는 시간은 어떻게 계산되나요?**
A. 일간 기록과 동일한 방식으로, (종료시간 - 시작시간) - 점심시간 - 일시정지시간으로 계산됩니다.

**Q. 과거 작업에 메모를 추가할 수 있나요?**
A. 네, 프로젝트 분석 페이지에서 세션 드릴다운을 열고 비고 열을 클릭하면 언제든 메모를 추가/수정할 수 있습니다.

**Q. 인터럽트 키워드는 저장되나요?**
A. 현재 버전에서는 페이지를 벗어나면 키워드가 초기화됩니다. 향후 버전에서 키워드 프리셋 저장 기능이 추가될 예정입니다.

**Q. 기존 주간 일정이나 일간 기록과 데이터가 다른가요?**
A. 아닙니다. 모든 페이지가 동일한 `logs` 데이터를 사용합니다. 집계 방식과 표시 형태만 다릅니다.
