## 1. 시작하기

v0.20.1에서는 커스텀 카테고리의 시간관리 불러오기 시 발생하던 머지 문제가 수정되었습니다.

## 2. 커스텀 카테고리 개선 사항

### 개요
사용자가 추가한 커스텀 카테고리가 시간관리 표로 불러오기할 때 정확하게 분리되어 표시됩니다.

### 이전 버전의 문제
- 시스템에 등록되지 않은 커스텀 카테고리가 모두 '9999'(기타) 코드로 변환됨
- 서로 다른 커스텀 카테고리가 하나로 머지되어 표시됨
- 프로젝트별, 카테고리별 시간 집계가 부정확함

### v0.20.1에서의 개선
- 각 커스텀 카테고리가 고유한 코드를 가짐
- 프로젝트별, 카테고리별로 정확히 분리됨
- 시간 집계가 정확해짐

## 3. 시간관리 불러오기 개선

### 사용 방법
1. 시간관리 페이지로 이동합니다
2. "일간 타이머에서 불러오기" 버튼을 클릭합니다
3. 해당 날짜의 타이머 로그가 시간관리 표로 변환됩니다

### 개선된 동작
커스텀 카테고리를 사용한 작업들이 각각 별도의 행으로 정확히 분리됩니다.

#### 예시: 이전 버전
```
프로젝트A - 커스텀작업1 (카테고리: 9999-기타) - 60분
프로젝트A - 커스텀작업2 (카테고리: 9999-기타) - 30분
→ 하나로 머지되어 90분으로 표시될 수 있음
```

#### 예시: v0.20.1
```
프로젝트A - 커스텀작업1 (카테고리: 커스텀작업1-커스텀작업1) - 60분
프로젝트A - 커스텀작업2 (카테고리: 커스텀작업2-커스텀작업2) - 30분
→ 각각 별도의 행으로 정확히 분리됨
```

## 4. 카테고리 처리 방식

### 시스템 카테고리
시스템에 미리 등록된 카테고리는 고유한 코드가 할당됩니다.

| 카테고리명 | 카테고리 코드 |
|-----------|--------------|
| 기획 | 01 |
| 개발 | 13 |
| 회의 | 05 |
| 분석 | 17 |
| 테스트 | 14 |
| ... | ... |

### 커스텀 카테고리
사용자가 추가한 카테고리는 **카테고리명이 곧 카테고리 코드**로 사용됩니다.

| 카테고리명 | 카테고리 코드 |
|-----------|--------------|
| 내 프로젝트 작업 | 내 프로젝트 작업 |
| 긴급 대응 | 긴급 대응 |
| 특별 업무 | 특별 업무 |

### 장점
- 각 커스텀 카테고리가 고유하게 식별됨
- 프로젝트별, 카테고리별 시간 집계가 정확해짐
- 엑셀 Export 시 데이터가 올바르게 분리됨

## 5. FAQ

**Q: 기존에 등록된 커스텀 카테고리 데이터는 어떻게 되나요?**
A: 기존 데이터는 변경되지 않습니다. 앞으로 새로 불러오는 데이터부터 개선된 방식으로 처리됩니다.

**Q: 시스템 카테고리와 같은 이름의 커스텀 카테고리를 만들면?**
A: 시스템 카테고리가 우선 적용됩니다. 예를 들어 "개발"이라는 커스텀 카테고리를 만들면 시스템의 "개발"(코드 13)으로 처리됩니다.

**Q: 커스텀 카테고리 코드가 너무 길면 문제가 되지 않나요?**
A: 시스템 내부적으로만 사용되며, 사용자에게는 카테고리명으로 표시되므로 문제없습니다.

**Q: 엑셀 Export 시 커스텀 카테고리는 어떻게 표시되나요?**
A: 카테고리 코드 열에 커스텀 카테고리명이 표시되고, 카테고리명 열에도 동일한 이름이 표시됩니다.

**Q: 이미 '9999'로 머지된 데이터는 어떻게 분리하나요?**
A: 시간관리 표에서 수동으로 행을 추가하고 시간을 분배하거나, 해당 날짜의 타이머 로그를 다시 불러오면 됩니다. (기존 행은 "삭제" 버튼으로 제거 가능)

**Q: 커스텀 카테고리를 나중에 삭제하면 어떻게 되나요?**
A: 시간관리 표에 이미 등록된 데이터는 유지되지만, 해당 카테고리명이 카테고리 코드로 그대로 남아있습니다.
