## 1. 업데이트 요약

v0.10.5에서는 **다크모드에서 테마색 가시성**이 크게 개선되었습니다.

### 주요 변경사항

| 변경 전 (v0.10.4 이하) | 변경 후 (v0.10.5) |
|---|---|
| 다크모드에서 어두운 테마색이 안 보임 | 자동으로 밝기 보정 |
| 색상별로 가시성이 불균일 | 모든 색상이 최소 45% 밝기 보장 |
| 일부 컴포넌트만 보정 | 모든 테마색 사용처에서 일관 적용 |

## 2. 다크모드 색상 자동 보정

### 2.1 자동 보정이란?

다크모드에서 배경이 어두워지면, 기존의 어두운 테마색(예: 네이비, 포레스트, 브라운 등)이 잘 보이지 않는 문제가 있었습니다.

v0.10.5부터는 **색상의 밝기(Lightness)가 45% 미만**이면 자동으로 밝게 조정됩니다.

### 2.2 보정 적용 범위

다음 모든 곳에서 보정된 테마색이 일관되게 적용됩니다:

| 위치 | 적용 항목 |
|---|---|
| **설정 페이지** | 팔레트 프리셋 미리보기, 대표 색상(Primary), 커스텀 팔레트 색상 |
| **간트 차트** | 작업 막대 배경색, 프로젝트 칩 색상 |
| **프리셋 패널** | 프리셋 항목 색상 표시 |
| **탭 & 버튼** | 선택된 탭 색상, 버튼 hover 색상 |

### 2.3 색조(Hue) 유지

보정 시 색상의 **색조는 그대로 유지**됩니다:
- 네이비(#264653) → 밝은 네이비(#3d8799)
- 포레스트 그린 → 밝은 포레스트 그린
- 원래 선택한 색감은 유지되면서 가시성만 개선

### 2.4 라이트모드에서는?

라이트모드에서는 색상 보정이 적용되지 않습니다. 원본 색상 그대로 표시됩니다.

## 3. 자주 묻는 질문

### Q1. 다크모드에서 색상이 달라 보여요

정상입니다! 다크모드에서는 가시성 확보를 위해 어두운 색상이 자동으로 밝게 조정됩니다. 색조는 동일하게 유지됩니다.

### Q2. 원본 색상을 그대로 사용하고 싶어요

현재 버전에서는 자동 보정을 끌 수 없습니다. 밝은 색상 팔레트(핑크 & 퍼플, 파스텔 등)를 선택하시면 보정 없이 원본 색상이 적용됩니다.

### Q3. 커스텀 색상도 보정되나요?

네, 커스텀 팔레트의 색상도 다크모드에서 밝기가 45% 미만이면 자동 보정됩니다.

### Q4. 보정된 색상과 원본 색상의 차이가 궁금해요

설정 페이지에서 라이트/다크 모드를 전환하며 팔레트 미리보기를 확인하시면 차이를 볼 수 있습니다.
