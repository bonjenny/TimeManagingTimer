## 1. 시작하기

TimeKeeper는 업무 시간을 기록하고 관리하는 웹 애플리케이션입니다. 모든 데이터는 브라우저의 LocalStorage에 저장되므로 별도의 로그인이 필요 없습니다.

### 저장 용량 확인
화면 하단에서 현재 저장 용량을 확인할 수 있습니다:
```
현재 저장 용량: 123.4 KB (0.12 MB / 약 5MB 제한)
```

## 2. 주요 기능

### 건의사항 게시판
사용자가 앱 개선 아이디어나 버그 리포트를 작성할 수 있는 게시판입니다.

#### 관리자 상태
관리자는 게시글에 상태를 설정할 수 있습니다:

| 상태 | 색상 | 설명 |
|------|------|------|
| 검토중 | 주황색 | 건의사항을 검토 중 |
| 반려 | 빨간색 | 건의사항이 반려됨 |
| 완료 | 초록색 | 건의사항이 적용됨 (버전 표시) |

#### 관리자 상태 변경 방법
1. 게시글 목록에서 해당 게시글을 **우클릭**
2. 관리자 상태 변경 모달이 표시됨
3. 원하는 상태 버튼 클릭 (검토중/반려/완료/상태 삭제)
4. 관리자 비밀번호 입력
5. 확인 버튼 클릭

## 3. v0.12.7 변경사항

### 버그 수정: 검토중/반려 상태 변경

**문제**
- 게시글을 "검토중" 또는 "반려" 상태로 변경하려고 하면 오류 발생

**해결**
- Firestore 데이터 처리 로직 개선으로 정상 동작

### 새 기능: 상태 삭제

관리자가 설정한 상태를 완전히 삭제할 수 있는 기능이 추가되었습니다.

**사용 방법**
1. 게시글 목록에서 해당 게시글을 **우클릭**
2. 관리자 상태 변경 모달에서 **"상태 삭제"** 버튼 클릭 (회색)
3. 관리자 비밀번호 입력
4. **"상태 삭제"** 확인 버튼 클릭
5. "상태가 삭제되었습니다." 토스트 알림 표시

**상태 삭제 후**
- 게시글에서 관리자 상태 라벨이 사라짐
- 완료 버전 정보도 함께 삭제됨

### 관리자 상태 버튼 UI

| 버튼 | 색상 | 동작 |
|------|------|------|
| 검토중 | 주황색 | 검토중 상태로 변경 |
| 반려 | 빨간색 | 반려 상태로 변경 |
| 완료 | 초록색 | 완료 상태로 변경 (버전 입력 필요) |
| 상태 삭제 | 회색 | 모든 상태 정보 삭제 |

## 4. FAQ

### Q: 검토중/반려 상태로 변경이 안 됐는데 이제 되나요?
네, v0.12.7에서 버그가 수정되어 모든 상태 변경이 정상 동작합니다.

### Q: 상태 삭제를 하면 어떻게 되나요?
게시글에서 관리자 상태 라벨(검토중/반려/완료)이 완전히 제거됩니다. 완료 버전 정보도 함께 삭제됩니다.

### Q: 상태 삭제 후 다시 상태를 설정할 수 있나요?
네, 다시 우클릭하여 새로운 상태를 설정할 수 있습니다.

### Q: 완료 상태에서 검토중으로 변경하면 버전 정보는 어떻게 되나요?
완료 → 검토중/반려로 변경하면 버전 정보는 자동으로 삭제됩니다.
