## 1. 시작하기

### 시간관리 탭 접근
- 상단 메뉴바에서 **"시간관리"** 탭 클릭
- 또는 단축키 **Alt + 5** 사용

## 2. 주요 기능

### 2.1 일간 타이머 데이터 불러오기
1. 날짜 선택기로 원하는 날짜 선택
2. **"일간 타이머에서 불러오기"** 버튼 클릭
3. 해당 날짜의 완료된 작업들이 자동으로 표에 추가됨
4. **같은 작업명의 로그들은 하나로 그룹화되어 시간이 합산됨**

### 2.2 인라인 편집
- 표의 각 셀을 클릭하면 편집 모드로 전환
- 수정 후 Enter 키 또는 다른 곳 클릭으로 저장
- ESC 키로 편집 취소

### 2.3 카테고리 코드
- 카테고리 코드 셀 클릭 시 드롭다운 메뉴 표시
- 코드 선택 시 카테고리명 자동 업데이트

#### 주요 카테고리 코드
| 코드 | 카테고리명 |
|------|------------|
| 01   | 기획       |
| 05   | 회의       |
| 10   | 질의응답   |
| 11   | 문서작업   |
| 13   | 개발       |
| 17   | 분석       |
| 9999 | 기타       |

전체 46개 카테고리 지원 (01~32, 101~109, 513, 9999)

### 2.4 새 행 추가 및 삭제
- **새 행 추가**: 빈 행 추가하여 수동 입력 가능
- **체크박스 선택 후 삭제**: 여러 행을 한 번에 삭제 가능

## 3. 시간관리 탭 사용법

### 기본 워크플로우

```
1. 날짜 선택
   ↓
2. "일간 타이머에서 불러오기" 클릭
   ↓
3. 자동 변환된 데이터 확인
   ↓
4. 필요한 셀 클릭하여 수정
   ↓
5. 카테고리 코드, 비고 등 보완
   ↓
6. 완성 (자동 저장됨)
```

### 컬럼 설명

| 컬럼명 | 설명 | 기본값/변환 규칙 |
|--------|------|------------------|
| **체크박스** | 다중 선택용 | - |
| **작업 (프로젝트코드)** | 프로젝트 코드 | TimerLog의 projectCode |
| **업무형** | 업무 유형 | "작업" (기본값) |
| **거래형(일정명)** | 업무기록명 (작업 제목) | TimerLog의 title |
| **카테고리 코드** | 숫자 코드 | 카테고리명 기반 자동 매핑 |
| **카테고리명** | 한글 카테고리명 | TimerLog의 category |
| **시간(분)** | 소요 시간(분) | startTime-endTime (일시정지 시간 제외) |
| **비고** | 메모 | TimerLog의 note |

### 날짜 네비게이션

- **← 버튼**: 이전 날짜로 이동
- **→ 버튼**: 다음 날짜로 이동
- **캘린더 아이콘**: 날짜 직접 선택
- **Today 아이콘**: 오늘로 이동 (오늘이 아닐 때만 표시)

### 데이터 독립성

시간관리 탭의 데이터는 일간 타이머와 **독립적으로 관리**됩니다:

- ✅ 시간관리 표를 수정해도 일간 타이머 원본 데이터는 변경되지 않음
- ✅ 시간관리 표에서 삭제해도 일간 타이머 기록은 유지됨
- ✅ 수동으로 새 행을 추가할 수 있음

## 4. FAQ

### Q1. 일간 타이머 데이터가 자동으로 표시되지 않나요?
A. 아니요. 날짜를 선택한 후 **"일간 타이머에서 불러오기"** 버튼을 클릭해야 데이터가 변환되어 표시됩니다.

### Q2. 중복 데이터가 추가되나요?
A. 아니요. 이미 불러온 데이터(original_log_id 기준)는 중복 방지되어 다시 불러와도 추가되지 않습니다.

### Q3. 카테고리 코드를 모르는데 어떻게 입력하나요?
A. 카테고리 코드 셀을 클릭하면 드롭다운 메뉴가 나타나며, 코드와 카테고리명을 함께 보여줍니다.

### Q4. 시간관리 표 데이터는 어디에 저장되나요?
A. 브라우저의 IndexedDB에 저장되며, 일간 타이머 데이터와는 별도로 관리됩니다.

### Q5. 편집 중 실수했을 때 취소할 수 있나요?
A. 예, ESC 키를 누르면 편집을 취소할 수 있습니다.
