# TimeKeeper v0.16.2 사용설명서

## 목차
1. 시작하기
2. 주요 변경사항
3. 데이터 저장 방식
4. FAQ

## 1. 시작하기

v0.16.2는 IndexedDB 데이터 로딩 안정성을 개선한 패치입니다.

### 기존 사용자
- 별도의 조치가 필요 없습니다.
- 이전 버전에서 새로고침 시 데이터가 보이지 않던 문제가 해결되었습니다.
- IndexedDB에 이미 저장된 데이터는 그대로 유지됩니다.

### 새 사용자
- 변경 사항 없이 바로 사용할 수 있습니다.

## 2. 주요 변경사항

### 데이터 로딩 안정성 개선
- 페이지 새로고침 시 모든 업무 기록이 정상적으로 표시됩니다.
- IndexedDB 데이터가 완전히 로드된 후에만 화면이 렌더링되어, 빈 화면이 나타나는 문제가 해결되었습니다.
- IndexedDB 접근 실패 시에도 앱이 정상 동작합니다 (에러 로그 출력).

## 3. 데이터 저장 방식

### 저장 위치
- 모든 데이터는 브라우저의 IndexedDB에 저장됩니다.
- 서버로 전송되는 데이터는 없습니다 (건의사항 게시판 제외).

### 데이터 백업
- 설정 페이지에서 "데이터 내보내기 (JSON)" 버튼으로 전체 데이터를 백업할 수 있습니다.
- 백업 파일은 `timekeeper-backup-YYYY-MM-DD.json` 형식으로 다운로드됩니다.

### 데이터 복원
- 설정 페이지에서 "데이터 가져오기" 버튼으로 백업 파일을 복원할 수 있습니다.
- 가져오기 시 기존 데이터가 덮어씌워지므로 주의하세요.

### 데이터 초기화
- 설정 페이지에서 "모든 데이터 초기화" 버튼으로 전체 데이터를 삭제할 수 있습니다.
- IndexedDB와 인메모리 캐시가 모두 초기화됩니다.

## 4. FAQ

### Q: 기존 데이터가 사라졌나요?
A: 아닙니다. 앱을 처음 열 때 기존 localStorage 데이터가 자동으로 IndexedDB로 복사됩니다.

### Q: 새로고침하면 데이터가 사라지는 문제는 해결되었나요?
A: 네. v0.16.2에서 Dynamic Import 방식을 도입하여 IndexedDB 데이터가 완전히 로드된 후에만 화면이 표시되도록 수정했습니다.

### Q: 브라우저 데이터를 삭제하면 어떻게 되나요?
A: "쿠키 및 사이트 데이터 삭제"를 하면 IndexedDB도 함께 삭제될 수 있습니다. 중요한 데이터는 반드시 "데이터 내보내기"로 백업하세요.

### Q: 저장 용량 제한이 있나요?
A: IndexedDB는 브라우저와 디스크 여유 공간에 따라 수백MB~수GB까지 저장 가능합니다. 기존 5MB 제한이 사라졌습니다.

### Q: 이전 버전의 백업 파일을 가져올 수 있나요?
A: 네. v0.14.x 이하에서 내보낸 JSON 백업 파일도 정상적으로 가져오기할 수 있습니다.
