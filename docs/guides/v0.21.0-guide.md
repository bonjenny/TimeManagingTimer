## 1. 시작하기

v0.21.0에서는 커스텀 카테고리 처리가 크게 개선되었습니다. 더 이상 커스텀 카테고리가 하나로 머지되지 않고, 프로젝트별, 카테고리별로 정확하게 분리됩니다.

## 2. 커스텀 카테고리 독립 처리

### 개요
사용자가 추가한 커스텀 카테고리가 시간관리 표로 불러올 때 독립적으로 처리됩니다.

### 카테고리 코드 부여 규칙

#### 시스템 카테고리 (코드맵에 등록된 카테고리)
시스템에 미리 등록된 카테고리는 고유한 코드가 할당됩니다.

| 카테고리명 | 카테고리 코드 |
|-----------|--------------|
| 기획 | 01 |
| 회의 | 05 |
| 개발 | 13 |
| 테스트 | 14 |
| 분석 | 17 |

#### 커스텀 카테고리 (사용자 추가 카테고리)
코드맵에 없는 카테고리는 **카테고리명이 곧 카테고리 코드**로 사용됩니다.

| 카테고리명 | 카테고리 코드 |
|-----------|--------------|
| 긴급 대응 | 긴급 대응 |
| 특별 프로젝트 | 특별 프로젝트 |
| 내 작업 | 내 작업 |

### 장점
- 각 커스텀 카테고리가 고유하게 식별됨
- 더 이상 모든 커스텀 카테고리가 '9999'(기타)로 통합되지 않음
- 프로젝트별, 카테고리별 시간 집계가 정확해짐

## 3. 프로젝트+카테고리+타이틀 그룹화

### 개요
v0.21.0부터 타이머 로그를 시간관리 표로 변환할 때 **프로젝트 + 카테고리 + 타이틀** 조합으로 그룹화됩니다.

### 변경 사항
- **이전 버전**: 타이틀만으로 그룹화
- **v0.21.0**: 프로젝트 + 카테고리 + 타이틀 조합으로 그룹화

### 그룹화 예시

#### 이전 버전 (타이틀만 그룹화)
```
로그:
1. 프로젝트A - 회의 (카테고리: 개발) - 30분
2. 프로젝트B - 회의 (카테고리: 회의) - 45분
3. 프로젝트A - 회의 (카테고리: 회의) - 60분

결과:
- 회의 - 135분 (모두 하나로 머지됨)
```

#### v0.21.0 (프로젝트+카테고리+타이틀 그룹화)
```
로그:
1. 프로젝트A - 회의 (카테고리: 개발) - 30분
2. 프로젝트B - 회의 (카테고리: 회의) - 45분
3. 프로젝트A - 회의 (카테고리: 회의) - 60분

결과:
- 프로젝트A - 개발 - 회의 - 30분
- 프로젝트B - 회의 - 회의 - 45분
- 프로젝트A - 회의 - 회의 - 60분
(각각 별도 행으로 분리됨)
```

### 장점
- 같은 타이틀이어도 프로젝트나 카테고리가 다르면 별도로 관리 가능
- 프로젝트별, 카테고리별 시간 집계가 정확해짐
- 엑셀 Export 시 데이터가 명확하게 분리됨

## 4. 시간관리 불러오기 개선

### 사용 방법
1. 시간관리 페이지로 이동합니다
2. "일간 타이머에서 불러오기" 버튼을 클릭합니다
3. 해당 날짜의 타이머 로그가 시간관리 표로 변환됩니다

### 개선된 동작
프로젝트, 카테고리, 타이틀이 모두 고려되어 정확하게 분리됩니다.

## 5. 사용 예시

### 예시 1: 커스텀 카테고리 사용

**타이머 로그:**
```
09:00-10:30 프로젝트A - "데이터 분석" (카테고리: 긴급 대응) - 90분
11:00-12:00 프로젝트A - "데이터 분석" (카테고리: 분석) - 60분
14:00-15:30 프로젝트B - "데이터 분석" (카테고리: 긴급 대응) - 90분
```

**시간관리 표 (v0.21.0):**
| 작업 | 거래형(일정명) | 카테고리 코드 | 카테고리명 | 시간(분) |
|------|-------------|-------------|----------|---------|
| 프로젝트A | 데이터 분석 | 긴급 대응 | 긴급 대응 | 90 |
| 프로젝트A | 데이터 분석 | 17 | 분석 | 60 |
| 프로젝트B | 데이터 분석 | 긴급 대응 | 긴급 대응 | 90 |

### 예시 2: 같은 타이틀, 다른 프로젝트/카테고리

**타이머 로그:**
```
09:00-10:00 프로젝트A - "회의" (카테고리: 회의) - 60분
11:00-12:00 프로젝트A - "회의" (카테고리: 개발) - 60분
14:00-15:00 프로젝트B - "회의" (카테고리: 회의) - 60분
```

**시간관리 표 (v0.21.0):**
| 작업 | 거래형(일정명) | 카테고리 코드 | 카테고리명 | 시간(분) |
|------|-------------|-------------|----------|---------|
| 프로젝트A | 회의 | 05 | 회의 | 60 |
| 프로젝트A | 회의 | 13 | 개발 | 60 |

```
프로젝트B - 회의 - 05 - 회의 - 180분 (잘못 머지됨)
```

## 6. FAQ

**Q: 기존 데이터는 어떻게 되나요?**
A: 기존 시간관리 표 데이터는 변경되지 않습니다. 앞으로 새로 불러오는 데이터부터 개선된 방식으로 처리됩니다.

**Q: 커스텀 카테고리를 나중에 삭제하면?**
A: 시간관리 표에 이미 등록된 데이터는 유지되지만, 해당 카테고리명이 카테고리 코드로 그대로 남아있습니다.

**Q: 엑셀 Export 시 커스텀 카테고리는 어떻게 표시되나요?**
A: 카테고리 코드 열에 커스텀 카테고리명이 표시되고, 카테고리명 열에도 동일한 이름이 표시됩니다.

**Q: 같은 타이틀, 같은 프로젝트, 같은 카테고리의 여러 로그는?**
A: 여전히 하나의 행으로 합쳐지며, 총 시간만 합산됩니다.

**Q: 프로젝트가 없는 작업은?**
A: 프로젝트가 없어도 카테고리와 타이틀 조합으로 그룹화됩니다. 빈 문자열이 프로젝트 코드로 사용됩니다.

**Q: 카테고리가 없는 작업은?**
A: 카테고리가 없으면 '기타' 카테고리로 처리되며, 코드는 '9999'가 됩니다.

**Q: 이전에 '9999'로 머지된 데이터를 분리할 수 있나요?**
A: 시간관리 표에서 수동으로 행을 추가하고 시간을 분배하거나, 해당 날짜의 타이머 로그를 다시 불러오면 됩니다. 기존 행은 "삭제" 버튼으로 제거할 수 있습니다.

**Q: 시스템 카테고리와 같은 이름의 커스텀 카테고리를 만들면?**
A: 시스템 카테고리가 우선 적용됩니다. 예를 들어 "개발"이라는 커스텀 카테고리를 만들면 시스템의 "개발"(코드 13)로 처리됩니다.

**Q: 그룹화 키는 어디서 확인할 수 있나요?**
A: 그룹화 키는 내부적으로만 사용되며, 사용자에게는 프로젝트명, 카테고리명, 타이틀로 각각 표시됩니다.
