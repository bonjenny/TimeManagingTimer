## 1. 시작하기

TimeKeeper는 업무 시간을 기록하고 관리하는 웹 애플리케이션입니다. 모든 데이터는 브라우저의 LocalStorage에 저장되므로 별도의 로그인이 필요 없습니다.

### 저장 용량 확인
화면 하단에서 현재 저장 용량을 확인할 수 있습니다:
```
현재 저장 용량: 123.4 KB (0.12 MB / 약 5MB 제한)
```

## 2. 주요 기능

### 타이머 상태
타이머(세션)는 4가지 상태를 가집니다:

| 상태 | 설명 | UI 표시 |
|------|------|---------|
| IDLE | 타이머 시작 전 | - |
| RUNNING | 작업 진행 중 | "진행 중" 칩 |
| PAUSED | 일시정지 상태 | 종료 시간(주황색) |
| COMPLETED | 작업 완료 | 종료 시간(기본색) |

### 일시정지(PAUSED) 발생 조건
1. 사용자가 일시정지 버튼 클릭
2. 다른 작업 시작 시 기존 작업이 자동으로 PAUSED
3. `Alt + S` 단축키로 일시정지/재개

### 완료 처리
- **완료 버튼**: 그룹의 모든 세션을 COMPLETED로 변경
- **완료 취소**: COMPLETED 세션을 PAUSED로 변경

## 3. v0.12.3 변경사항

### 간트차트 개선
- **우클릭**: 세션을 추가하지 않고 컨텍스트 메뉴만 표시
- **짧은 드래그**: 5분 미만 드래그 시 무시 (실수 방지)

### 완료 버튼 수정
- 미완료 작업도 완료 버튼으로 한 번에 완료 처리 가능
- 진행 중인 작업(활성 타이머)은 완료 버튼 비활성화

### 저장 용량 표시
- 화면 하단에서 현재 LocalStorage 사용량 확인 가능
- 약 5MB 제한 대비 사용량을 KB/MB 단위로 표시

### 프리셋 패널 개선
- 텍스트가 길면 자동으로 말줄임(...) 표시
- 아이콘 버튼과 텍스트가 겹치지 않도록 개선

## 4. FAQ

### Q: 일시정지와 완료의 차이는?
- **일시정지(PAUSED)**: 임시 중단. 재개하거나 다시 시작 가능
- **완료(COMPLETED)**: 작업 종료. 새 세션으로만 다시 시작 가능

### Q: 저장 용량이 부족하면?
- 설정 > 데이터 관리에서 오래된 데이터 삭제
- 휴지통 비우기로 삭제된 데이터 정리
- JSON 파일로 백업 후 오래된 데이터 제거

### Q: 간트차트에서 실수로 세션을 추가했어요
- 5분 미만 드래그는 무시되므로 짧은 클릭/드래그는 세션이 생성되지 않습니다
- 잘못 추가된 세션은 우클릭 > 삭제로 제거 가능

### Q: 완료된 작업을 다시 시작하려면?
- 완료 버튼(체크 아이콘)을 다시 클릭하면 완료 취소
- 또는 재생 버튼으로 새 세션 시작
