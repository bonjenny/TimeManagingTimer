## 1. 시작하기

TimeKeeper는 업무 시간을 기록하고 관리하는 웹 애플리케이션입니다. 모든 데이터는 브라우저의 LocalStorage에 저장되므로 별도의 로그인이 필요 없습니다.

### 저장 용량 확인
화면 하단에서 현재 저장 용량을 확인할 수 있습니다:
```
현재 저장 용량: 123.4 KB (0.12 MB / 약 5MB 제한)
```

## 2. 주요 기능

### 타이머 상태
타이머(세션)는 4가지 상태를 가집니다:

| 상태 | 설명 | UI 표시 |
|------|------|---------|
| IDLE | 타이머 시작 전 | - |
| RUNNING | 작업 진행 중 | "진행 중" 칩 |
| PAUSED | 일시정지 상태 | 종료 시간(주황색) |
| COMPLETED | 작업 완료 | 종료 시간(기본색) |

### 소요시간 계산
- **기본 계산**: `종료시간 - 시작시간 - 일시정지시간`
- **점심시간 제외**: 설정에서 점심시간을 지정하면 해당 시간은 자동 제외
- **비정상 데이터 처리**: 일시정지시간이 전체 작업시간보다 크면 무시

## 3. v0.12.4 변경사항

### 소요시간 계산 개선
일부 로그에서 소요시간이 0분으로 잘못 표시되는 문제가 수정되었습니다.

**문제 상황**
- 작업시간: 14:16 ~ 14:40 (24분)
- 일시정지시간: 28분 (비정상 데이터)
- 표시 결과: 0분 ❌

**수정 후**
- 비정상 데이터 감지: 일시정지시간 > 작업시간
- 비정상 데이터 무시: 일시정지시간을 0으로 처리
- 표시 결과: 24분 ✓

### 영향 범위
- 작업 이력 목록 (TimerList)
- 주간 일정 (WeeklySchedule)
- 활성 타이머 누적시간 (ActiveTimer)

## 4. FAQ

### Q: 소요시간이 0분으로 표시돼요
v0.12.4에서 수정되었습니다. 브라우저를 새로고침하면 정상 표시됩니다.

### Q: 일시정지 시간이 비정상적으로 저장된 이유는?
타이머 상태 전환 과정에서 데이터가 잘못 저장되는 경우가 있었습니다. v0.12.3에서 근본 원인이 수정되었으며, v0.12.4에서는 기존 비정상 데이터도 정상 표시되도록 처리했습니다.

### Q: 데이터를 직접 수정해야 하나요?
아니요. v0.12.4에서는 비정상 데이터를 자동으로 감지하고 무시하므로 데이터 수정이 필요 없습니다.

### Q: 백업 파일의 비정상 데이터는?
백업 파일 복원 시에도 비정상 데이터가 자동으로 처리됩니다.
