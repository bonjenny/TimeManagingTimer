## 1. 프리셋 작업 일별 고유 관리

프리셋으로 매일 반복 추가하는 작업(예: "관리업무 문서작업")의 누적시간이 날짜를 넘어 계속 쌓이는 것을 방지하는 기능입니다.

### 기능 요약
- 프리셋으로 시작한 작업에 내부적으로 날짜 기반 고유 키를 자동 부여
- 같은 제목이라도 다른 날짜의 작업은 누적시간이 별도로 계산됨
- 작업 제목은 변경되지 않으며, 사용자에게 보이는 UI 변화 없음

## 2. 설정 방법

1. **설정** 페이지로 이동합니다
2. **프리셋 작업 일별 고유 관리** Switch를 찾습니다
3. Switch를 켜면 기능이 활성화됩니다
4. 설정은 자동으로 저장됩니다

## 3. 작동 방식

### 기능 비활성 시 (기존 동작)
```
2/24: "문서작업" 프리셋 시작 → 누적시간: 2/24까지 "문서작업" 전체 합산
2/25: "문서작업" 프리셋 시작 → 누적시간: 2/24 + 2/25 합산
2/26: "문서작업" 프리셋 시작 → 누적시간: 2/24 + 2/25 + 2/26 합산
```

### 기능 활성 시
```
2/24: "문서작업" 프리셋 시작 → 누적시간: 2/24의 "문서작업"만 합산
2/25: "문서작업" 프리셋 시작 → 누적시간: 2/25의 "문서작업"만 합산
2/26: "문서작업" 프리셋 시작 → 누적시간: 2/26의 "문서작업"만 합산
```

### 적용 범위
| 영역 | 반영 여부 |
|------|-----------|
| 일간 타이머 (총시간 = 누적시간) | O |
| 주간일정 (누적시간 모드) | O |
| 작업 다시시작 | O (원래 dailyGroupKey 유지) |
| 수동 입력 작업 | X (수동 입력은 영향 없음) |

## 4. FAQ

### Q: 기존에 기록된 작업에도 적용되나요?
A: 아닙니다. 이 기능은 설정 활성화 이후 프리셋으로 시작한 새 작업에만 적용됩니다. 기존 로그는 기존 방식대로 동작합니다.

### Q: 수동으로 입력한 작업에도 적용되나요?
A: 아닙니다. 이 기능은 프리셋 패널에서 시작한 작업에만 적용됩니다. 수동 입력이나 다른 방식으로 시작한 작업은 기존처럼 동작합니다.

### Q: 작업을 "다시 시작"하면 어떻게 되나요?
A: 다시 시작하면 원래 작업의 날짜 기반 키가 그대로 유지됩니다. 즉, 2/24에 시작한 프리셋 작업을 다시 시작하면 2/24 그룹으로 계속 집계됩니다.

### Q: 주간일정의 "당일시간" 모드와는 어떤 관계인가요?
A: "당일시간" 모드는 해당 날짜의 작업 시간만 표시하는 표시 방식 변경이고, "프리셋 일별 고유 관리"는 누적시간의 계산 범위 자체를 날짜별로 분리하는 기능입니다. 두 기능은 독립적으로 동작합니다.
