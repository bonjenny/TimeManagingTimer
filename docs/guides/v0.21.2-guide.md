## 1. 시작하기

v0.21.2에서는 주간일정 페이지에 관리업무 필터 기능이 추가되었습니다. 이 기능을 통해 관리업무를 제외하고 실제 업무만 집중해서 볼 수 있습니다.

## 2. 주간일정 관리업무 필터 기능

### 개요
주간일정 페이지 상단에 있는 토글 버튼을 통해 관리업무를 표시하거나 제외할 수 있습니다.

### 주요 기능
- **토글 버튼**: "전체보기" ↔ "관리업무 제외" 상태 전환
- **자동 필터링**: 관리업무로 분류된 항목이 주간 일정 목록과 복사 미리보기에서 모두 제외됨
- **상태 저장**: 토글 상태가 localStorage에 저장되어 다음 방문 시에도 유지됨

## 3. 사용 방법

### 기본 사용법

1. **주간일정 페이지 이동**
   - 메뉴에서 "주간 일정"을 선택합니다

2. **필터 토글 버튼 확인**
   - 페이지 상단에 "전체보기" 버튼이 표시됩니다
   - 이 버튼을 클릭하면 "관리업무 제외" 버튼으로 변경됩니다

3. **관리업무 제외하기**
   - "전체보기" 버튼을 클릭합니다
   - 버튼이 "관리업무 제외"로 변경되고, 관리업무가 목록에서 제외됩니다

4. **전체보기로 복원**
   - "관리업무 제외" 버튼을 다시 클릭합니다
   - 버튼이 "전체보기"로 변경되고, 모든 업무가 표시됩니다

### 사용 예시

#### 전체보기 모드
```
월요일 (3/17)
├─ [A25_05591] 관리업무
│  └─ > 주간회의
├─ [A26_00001] 프로젝트A
│  └─ > API 개발
└─ [A26_00002] 프로젝트B
   └─ > UI 구현
```

#### 관리업무 제외 모드
```
월요일 (3/17)
├─ [A26_00001] 프로젝트A
│  └─ > API 개발
└─ [A26_00002] 프로젝트B
   └─ > UI 구현
```

## 4. 관리업무 분류 기준

다음 조건 중 하나라도 해당되면 관리업무로 분류됩니다:

### 1. 프로젝트 코드 기준
- 프로젝트 코드가 `A25_05591`인 경우

### 2. 프로젝트명 기준
- 프로젝트명이 `관리업무`인 경우

### 예시
```
[A25_05591] 관리업무 → 필터링 대상 ✓
[A25_05591] 기타업무  → 필터링 대상 ✓
[A26_00001] 관리업무  → 필터링 대상 ✓
[A26_00002] 개발업무  → 필터링 대상 ✗
```

## 5. FAQ

**Q: 관리업무 제외 시 복사 미리보기에도 적용되나요?**
A: 네, 관리업무 제외 모드에서는 주간 일정 목록과 복사 미리보기 모두에서 관리업무가 제외됩니다.

**Q: 관리업무 분류 기준을 변경할 수 있나요?**
A: 현재 버전에서는 `A25_05591` 프로젝트 코드와 `관리업무` 프로젝트명이 고정 기준입니다. 향후 버전에서 사용자 지정 기능이 추가될 수 있습니다.

**Q: 필터 설정이 저장되나요?**
A: 네, 토글 상태가 localStorage에 저장되어 브라우저를 닫았다가 다시 열어도 이전 설정이 유지됩니다.

**Q: 다른 페이지에도 적용되나요?**
A: 아니요, 이 필터는 주간일정 페이지에만 적용됩니다. 다른 페이지(시간관리, 최근 업무 기록 등)에는 영향을 주지 않습니다.

**Q: 특정 프로젝트 제외 필터와 함께 사용할 수 있나요?**
A: 네, 관리업무 필터와 특정 프로젝트 제외 필터는 독립적으로 동작하며, 두 필터를 동시에 적용할 수 있습니다.

**Q: 관리업무가 없는데도 버튼이 보이나요?**
A: 네, 관리업무 유무와 관계없이 토글 버튼은 항상 표시됩니다. 관리업무가 없는 경우 필터를 활성화해도 화면에 변화가 없을 수 있습니다.

**Q: 엑셀 Export 시 필터가 적용되나요?**
A: 주간일정 페이지의 복사 기능을 사용할 때는 필터가 적용됩니다. 별도의 엑셀 Export 기능이 있다면 해당 기능의 설정을 따릅니다.

**Q: 관리업무를 여러 개 지정할 수 있나요?**
A: 현재는 프로젝트 코드 `A25_05591`과 프로젝트명 `관리업무` 기준만 지원합니다. 여러 프로젝트를 관리업무로 분류하려면 각 프로젝트의 이름을 `관리업무`로 설정하면 됩니다.

**Q: 필터를 초기화할 수 있나요?**
A: 토글 버튼을 클릭하여 원하는 상태로 전환하거나, 브라우저의 localStorage를 삭제하면 기본값(전체보기)으로 초기화됩니다.

**Q: 모바일에서도 동작하나요?**
A: 네, 반응형 디자인으로 구현되어 모바일 환경에서도 정상적으로 동작합니다.
