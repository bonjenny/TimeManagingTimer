# 11. 예약 일정 (Scheduled Task)

## 개요
미래 시간의 작업을 사전 등록하고, 해당 시간이 되면 현재 타이머를 자동으로 paused 처리 후 예약된 작업의 타이머를 자동 시작하는 기능.

## 핵심 기능

### 예약 작업 등록
- **간트차트 좌클릭 드래그**: 미래 시간대를 드래그하면 자동으로 SCHEDULED 상태로 생성
- **간트차트 우클릭 드래그**: 시간대와 관계없이 항상 SCHEDULED로 생성 (주황색 드래그 영역)
- **TimerInput 예약 모드**: 시계 아이콘(ScheduleIcon) 클릭 후 시작/종료 시간 입력하여 등록

### 자동 타이머 전환
- 5초 간격으로 SCHEDULED 로그의 startTime을 체크
- startTime에 도달하면 현재 activeTimer를 paused 처리 후 예약 작업을 RUNNING으로 전환
- scheduledEndTime에 도달하면 자동으로 COMPLETED 처리

### 시각적 구분
- 간트차트에서 점선 테두리 + 사선 패턴으로 표시
- 툴팁에 [예약] 라벨 표시
- 우클릭 드래그 시 주황색 점선 드래그 영역

### 컨텍스트 메뉴
- '지금 시작': 예약 시간 전에 즉시 타이머 시작
- '수정': 시간/제목/프로젝트 등 수정
- '삭제': 예약 취소

## 데이터 구조

### TimerStatus
`'SCHEDULED'` 상태 추가

### TimerLog 필드
- `scheduledEndTime?: number` - 예약 작업이 RUNNING으로 전환될 때 원래 endTime을 보존

## 필터링
- TimerList: SCHEDULED 로그 제외 (최근 업무 기록에 미표시)
- WeeklySchedule: SCHEDULED 로그 제외
- timeManagementConverter: SCHEDULED 로그 변환 대상에서 제외
